    SFusion

    发布时间 : 2023-11-15 17:01
    阅读 :

    提出的问题：
    当存在丢失的模式时，可用模式的数量在不同的情况下是不同的，这导致了N对1的融合问题。
    解决方法：
    为了解决这一问题，我们提出了一种基于自我注意的融合块SFusion。
    与预置公式或基于卷积的方法不同，所提出的块自动学习融合可用模式，而不合成或补零丢失的模式。具体地说，从上游处理模型中提取的特征表示被投影为令牌，并被馈送到自我注意模块以产生潜在的多模式关联。在此基础上，引入了一种通道注意机制来构建共享表征，并将其应用于下游决策模型。

    /*
    在医学图像处理和机器学习领域，特别是在使用深度学习模型如卷积神经网络（CNN）或者Transformer模型时，”特征表示”（feature representations）是指从数据中自动学习到的高维特征，这些特征捕捉了数据的重要信息，可以用于后续的分析或决策任务。

    当我们说”从上游处理模型中提取的特征表示被投影为tokens”时，这里的”tokens”通常指的是一种可以被模型进一步处理的数据格式。在深度学习中，尤其是在使用类似于Transformer的模型时，”token”这个术语经常被用来描述输入数据的离散单元。以下是这个过程的一般理解：

    1. 特征提取：上游处理模型（如CNN）会从原始图像中提取特征。这些特征通常是高维的，并且包含了图像的关键信息，比如边缘、纹理、形状等。

    2. 特征表示：提取的特征会被编码成一系列的特征向量，每个向量代表图像中的一部分或某些特性。这些向量就是特征表示。

    3. 投影为Tokens：在某些深度学习模型，尤其是Transformer模型中，输入需要被分割成一系列的小块或单元，这些单元被称为”tokens”。特征表示被转换（或投影）成这种格式，以便它们可以被模型接受和处理。在图像处理的上下文中，这可能意味着将特征向量转换为一系列可以被Transformer模型处理的序列。

    4. 模型处理：一旦特征表示被转换为tokens，它们就可以被输入到下游模型中（如Transformer的编码器），在那里它们会被用于进一步的分析，比如分类、检测或分割任务。

    这个过程允许模型利用上游处理模型提取的复杂特征，并通过下游模型进行高级的理解和决策。在医学图像处理中，这种方法可以帮助提高疾病诊断的准确性，通过更好地理解和利用图像数据中的复杂模式。

    */

    在实际应用中，情态缺失是一种常见的情况。由于伪影和不同的病人情况，医学图像可能会丢失。为了处理这一问题，一个直观的解决方案是在所有可能的可用模态子集上训练专用模型。然而，这些方法都是低效且耗时的。
    另一种方法是预测缺失的模式，并使用已完成的模式执行。但是，这些方法还需要为每个缺失情况建立额外的预测网络，而恢复数据的质量直接影响到性能。

    目前，现有融合策略可以大致分为三类：算数策略、选择策略和卷积策略。

    算数策略：可用模态的特征表示被算术函数合并，诸如平均、计算一阶矩和二阶矩或其他设计的公式。

    选择策略：从输入的相应位置处的值中选择融合表示的每个值。选择规则可以定义为最大、最小或基于概率。

    上述两种融合策略很容易扩展到各种数据缺失的情况，但它们的融合操作是硬编码的。

    卷积策略：卷积融合网络自动学习如何融合这些特征表示，这有利于利用多个模态之间的相关性。然而，如图1(C)所示，该融合策略需要恒定数量的数据来满足卷积网络中输入通道的要求。因此，它必须通过粗略地补零或用类似的模式替换来模拟丢失的数据，这不可避免地在计算中引入偏差并导致性能下降。

    Transform在计算机视觉领域取得了成功，证明了自我注意机制具有捕捉图像表征的潜在相关性的能力。

    如图1(D)所示，SFusion可以处理任意数量的输入数据，而不是固定其数量。此外，SFusion是一种基于学习的融合策略，由两部分组成：相关性提取(CE)模块和通道注意(MA)模块。

    ——在CE模块中，从可用通道中提取的特征表示被投影为令牌，并被馈送到自我关注层以学习多通道相关性。
    ——基于这些相关性，提出了一种用于在MA模块中生成权重图的模式Softmax函数。最后，通过融合不同的输入和权重映射建立共享的特征表示。

    /*
    我们使用SHL[22]数据集对活动识别进行定性和定量的性能评估，并使用BraTS2020[1]数据集对脑瘤分割进行评估。
    */


    由两个模块组成：相关提取(CE)模块和通道注意(MA)模块。

    每一层包括多头注意(MHA)块和完全连接的前馈网络(FFN)。在每个块之前应用层归一化(LN)。

    输入特征表达保持不变。它使主干网络(上游处理模型和下游决策模型)能够增强其对来自不同模式的信息进行编码和解码的能力，而不是依赖于特定模式。这对于变量多峰数据分析是至关重要的。

    为了防止过度拟合，在训练期间应用了两种数据增强技术(在[−10◦，10◦]中随机翻转轴和以随机角度旋转)。

    SFusion比较了两种方法EmbraceNet和GFF
    前者采用选择策略，后者采用卷积策略

    作为一种依赖于数据的融合策略，SFusion能够自动学习不同模态之间的潜在相关性，并建立共享的特征表示。整个融合过程是基于可用的数据，而不是模拟丢失的模式。此外，SFusion还兼容任何一种上游处理模型和下游决策模型，使其普遍适用于不同的任务。
